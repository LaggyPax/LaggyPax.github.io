LOB-GATv2: An Evolution

LOB-GATv2 · Version 1

Can the Order Book Be Learned?
A First Proof of Structure

Oct 2025 — Dec 2025
AAPLLOBSTERGATv2 Hurst ExponentProof of Concept
+
The Question

Before writing a single line of model code, I needed to answer a prior question: is there learnable structure in limit order book data at all? A model trained on noise is worse than no model — it gives false confidence. So the first version of this project was less about architecture and more about justification.

I started with a Hurst exponent analysis on AAPL tick data in LOBSTER format. The result — H ≈ 0.76 — indicated strong long-memory persistence. This is not random walk territory. The order book has memory, and that memory is statistically detectable. That single number gave me the licence to proceed.

The Core Idea

The representational insight of V1: an order book at any tick is not a flat vector — it is a relational structure. Bid and ask sides mirror each other. Price levels are ordered. The spread between them carries information. So rather than feeding raw LOB data into a sequential model, I asked: what if we treat the 20 price-volume points as nodes in a graph, connected by their structural relationships?

I built a GATv2-based graph attention network. GATv2 (Brody et al., ICLR 2022) makes attention truly dynamic — the score between two nodes depends on both their features jointly, not independently. For order book data, where the relationship between a bid and an ask level is fundamentally pairwise, this distinction matters. V1 used both within-tick spatial edges and cross-tick temporal edges, with log-difference input features for stationarity.

What Worked · What Failed
✓ Worked

The model learned. Validation MSE came in well below the naive baseline — confirming that graph-structured representations of the LOB carry real predictive signal. The model concentrated attention on BID level 3, suggesting mid-book liquidity is structurally significant.

✗ Failed / Learned

Train/validation leakage from overlapping sequence windows near the chronological split inflated early results. Fixed with a gap buffer. Embeddings initially appeared frozen — confirmed correct as registered buffers, not trainable parameters.

LOB-GATv2 · Version 2

Broader Horizons.
Testing Generalisability on QQQ

Dec 2025 — Feb 2026
QQQCross-AssetFull LOB Shape MSE LossAWS EC2
+
The Question

V1 proved structure existed in AAPL. But a model that only works on one ticker is not a model — it is memorisation. V2 was about breadth: does the same architecture generalise? Can the same representational choices extract signal from a different instrument with different liquidity dynamics?

I moved to QQQ — an ETF with its own order book characteristics, distinct from a single-name equity. The expanded dataset also meant more serious infrastructure. I migrated training to AWS EC2 (Tesla T4, ap-southeast-2), which introduced its own operational challenges.

The Architecture Shift

V1 predicted a directional signal. V2 shifted to predicting the full next-tick order book shape: all 20 price-volume points across both sides. This is a harder task, but a more honest one. Directional prediction collapses a rich structure into a single bit. Full shape prediction forces the model to genuinely understand the book.

MSE over all 40 output values became the loss function. Side embeddings and level embeddings were added to give each node a structural identity — so the model knows not just what a node's value is, but where it sits in the book's hierarchy. A cosine annealing learning rate schedule was introduced for training stability.

What Worked · What Failed
✓ Worked

Generalisation held. The model found signal in QQQ with the same core architecture. Side and level embeddings meaningfully improved attention quality by giving each node structural identity beyond its raw feature values.

✗ Failed / Learned

Weight update deceleration was misread as a data insufficiency problem. It was the cosine annealing schedule doing its job. Lesson: understand your training dynamics before diagnosing your data.

LOB-GATv2 · Version 3 — Nemesis

Spatial Meets Temporal.
A Hierarchical Dual-Graph Architecture

Feb 2026 — Present
Hierarchical GNNWithin-Tick Spatial Cross-Tick TemporalDual GraphActive Research
+
The Question

V1 and V2 used cross-tick temporal edges as the primary graph signal. But at any single tick, the order book is also a spatial structure — bid levels relate to each other, ask levels relate to each other, and both sides interact across the spread. What happens if we model both simultaneously, in a hierarchical two-stage architecture?

I called this version Nemesis — not as a statement of confidence, but as a reminder that complexity has costs. Every new component is a new place for things to go wrong.

The Architecture

Nemesis is a hierarchical dual-graph system. Stage one applies GATv2 attention within each tick — treating the 20 LOB nodes as a spatial graph where edges encode structural relationships between price levels and across the bid-ask spread. This produces a tick-level embedding that captures the snapshot structure of the book at that moment.

Stage two applies GATv2 attention across ticks — where the long-memory signal from the Hurst analysis is most directly exploited. The two-stage design means the model first learns what the book looks like at a point in time, then learns how those snapshots evolve.

What Is Working · What Remains Open
✓ Progress

The hierarchical separation of spatial and temporal processing is architecturally clean. Early results suggest the within-tick stage improves the quality of representations fed into the temporal stage.

○ Open Questions

Does the spatial graph add genuine signal over pure temporal modelling, or noise? The ablation is in progress. Computational cost of dual-graph attention at scale is a real constraint.

On The Horizon

I am exploring an extension: using attention mechanisms to dynamically parameterise Kalman filter matrices (Q, R, F) conditioned on LOB sequence context — bridging differentiable state-space models with graph attention. This connects to the DKFAN literature and, to my knowledge, has not been applied to limit order book modelling.

About

About

I am an independent quantitative researcher based in Melbourne, working at the intersection of machine learning and market microstructure. Since October 2025, I have dedicated myself to understanding how graph attention networks can extract predictive structure from limit order book data.

My research focuses on meta-labelling and AI-driven tick data structuring, graph and video attention architectures for financial time series, and the behaviour of informed and uninformed traders as studied through machine learning. These interests are unified by a single conviction: market microstructure is not noise — it is legible, and the right representations make it so.

My academic background includes coursework at the University of Melbourne, and I have engaged with researchers across market microstructure and neuroeconomics. My practical background is as a self-taught builder — comfortable across the full stack from data pipeline to model training to cloud infrastructure.
